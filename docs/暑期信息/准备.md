### subagent

主智能体将复杂目标拆解，并调用拥有独立上下文窗口、特定模型配置或专属工具的子智能体。每个子智能体拥有独立的系统提示词（System Prompt）和执行环境。

上下文管理：单一模型处理极长上下文时，注意力的分布会稀释（Lost in the Middle）。子智能体通过 context isolation 保持任务专注度。

成本与延迟：主 Agent 可以用 GPT-4/Claude 3.5 负责规划，而子智能体可以使用更小、更快的模型（如 GPT-4o-mini 或 Llama 3 8B）执行简单任务。

容错性：单个子智能体失败不会导致整个系统崩溃，可以针对性地进行重试或人工干预。

思索：观察者agent

## SFT

### lora

全参数微调会调整预训练的所有参数，显存极高（参数量的4倍），所需数据量极大。理论上线高，适合复杂任务；但容易导致灾难性遗忘（模型之前的能力退化）。  

lora不训练所有参数，训练一个低秩的参数变化量矩阵，可以写为ba的形式。参数量约为原来的0.1%到1%。

在微调中，原参数会被冻结。具体而言，原参数会被标记，在反向传播时，会直接跳过被标记的张量，不计算该张量的梯度。初始化时，$A$用高斯初始化，$B$零初始化，保证最初$\Delta W$为零（训练起点与原模型完全相同）。

r较大时，可训练参数更多、显存占用更大、微调能力更强；但对应的，也会增加过拟合风险。可以在最初微调时采取较大r，在收敛后减小r或采用其它正则化方法。一般来说，可以先选取r=8作为基线。

## RL

on policy：运行模型得到数据，并直接通过数据更新参数。更新参数后重新运行得到数据。采集的数据只使用一次。  
off policy：运行模型得到一系列数据，再用这些数据更新参数。缺点是另一个专家模型的数据分布可能和当前有差异，使得当前策略便宜到错误方向。

参数：  
- S：状态空间
- A：动作空间
- s_t：t时刻状态
- R：s时执行a的奖励
- \gamma：折扣因子
- V：状态的期望折扣奖励
- Q：动作的期望折扣奖励
- A：Q - V，衡量动作好坏

普通RLHF需要四个模型：策略、冻结策略、reward、critic（价值）。

### TRPO

简单来说，新旧策略差异等于1/(1-\gamma)E[A]。忽略策略状态分布变化，动作由新策略采样得到。

$$
max L \\
L = J(\theta) + E[(\pi_{\theta'(a|s)}/(\pi_{\theta(a|s)})A(s,a)]
$$

在新旧策略差异一定的情况下（KL散度）max L。

进一步简化（泰勒展开）：

$$
\theta_{k+1} = max g^T(\theta' - \theta_k) \\
(\theta' - \theta_k)^TH(\theta' - \theta_k)/2 \le \delta
$$

g代表上式L的梯度，H表示策略平均KL散度的黑塞矩阵。

### PPO

TRPO计算比较复杂。

PPO分为PPO penalty和PPO clip两种变体。

- PPO clip
  - 计算TRPO的L（重要性采样乘A）
  - 计算clip(重要性采样，1 - $\epsilon$，1 + $\epsilon$)
  - L和clip取最小值

- PPO penalty
  - 计算TRPO的L
  - DL小于$\delta$/1.5（超参数），beta砍半；大于，乘二，否则不变

GAE用于计算A。

### DPO

RLHF中PPO/TRPO需要训练奖励模型。DPO不训练奖励模型，直接训练策略模型。只需要参考+策略模型。

策略模型是DPO需要训练的模型，后用在项目中的模型。策略模型的权重直接复制SFT阶段微调模型的权重。  
参考模型是策略模型的帮衬，其权重参数冻结不变。主要两个作用，其一协助其计算reward loss，其二计算kl正则项，防止其训练偏移初始SFT模型太远，由一个β参数控制。  

损失：

$$
-E[log\sigma(\beta log(chosen) - \beta log(reject))]
$$

- chosen：$\pi(y_w|x) / \pi_{ref}(y_w|x)$
- reject：$\pi(y_l|x) / \pi_{ref}(y_l|x)$

### GRPO

PPO的变种，省去了价值模型（计算状态回报）。通过生成一组回答得到平均值作为价值的估计。通过价值模型计算平均奖励。

适合计算资源受限，或任务存在客观、可程序化验证的奖励标准的场景 (如代码、数学)。

### DAPO

GRPO问题：

- Token 级别的 Clip 容易导致熵崩溃：模型很快收敛到少量固定答案，导致多样性和探索能力不足（熵崩溃）。
- Batch 采样中出现奖励极端化：部分样本的奖励可能全部为 1 或 0，从而产生「零梯度」问题，削弱训练信号。
- 长序列训练的梯度分布失衡：权重分布让极少数 token 的梯度占据主导，导致许多高质量的长序列样本被忽视。

## vllm

vllm重点优化了kv cache的显存占用，即将内存碎片等一系列系统方法提供给显存。之前的问题：  
1. 预分配过多（句子未达到max token）

## 机器学习基础

回归问题：建立一个从输入特征（自变量）到连续数值目标（因变量）的映射关系。

### 线性回归

假设输入与输出呈线性关系。  
最小二乘法：让残差平方和最小。beta=(x^Tx)-1X^Ty

### 决策树

对于一个信息，要计算出如何高度最小。用于分类和回归问题。

ID3：基于信息增益（信息熵）。先计算出每个类别的概率，算出总信息熵。将数据集按每个属性划分为k个子集，算出每个决策的信息增益，增益越大越靠近根。问题：偏好取值数目较多的规则。

C4.5：gain_ratio = gain/v。r取值越多惩罚越大。会采用启发式：先找出信息增益高的，再找出信息增益率高的。

CART：gini=1-sigma p^2。gini越小越靠近根。

### 聚类

k-means：预先决定k，并选取k个点。计算所有到k的距离，归为簇。然后计算每个簇的质心（所有点的算数平均），进行下一轮。
